eXtensible Trajectory File format Specification (extension .xtf or .xml)
--------------------------------------------------------------------------

The eXtensible Trajectory File format is designed to enable the interchange
of trajectories between various robotics software systems and robots. The
format is deliberately robot- and platform-agnostic, and the reference C++
and Python implementations do not use platform-specific types or objects.

Specifically, the XTF format is designed to store both trajectories
generated by a planner or trajectory generator, and trajectories recorded
from a robot or robot-like kinematic chain. These trajectories can be either
joint-space or pose-space data, and the format offers the necessary fields
to fully define either.

Shown below is an overview of the file format structure. See ./bin for
the example pose trajectory files used to test the XTF libraries.
--------------------------------------------------------------------------

<?xml version="1.0" encoding="utf-8" ?>
<trajectory uid="trajectory1">
  <info robot="PR2" generator="reference">
    <type traj_type="recorded" timing="timed" data_type="pose">
      <root_frame>

      </root_frame>
      <target_frame>

      </target_frame>
    </type>
    <type traj_type="generated" timing="untimed" data_type="joint">
      <joint_names>
        
      </joint_names>
    </type>
    <tags>

    </tags>
  </info>
  <states length="1">
    <state sequence="0" secs="0" nsecs="0">
      <desired>
        <position>

        </position>
        <velocity>

        </velocity>
        <acceleration>

        </acceleration>
      </desired>
      <actual>
        <position>

        </position>
        <velocity>

        </velocity>
        <acceleration>

        </acceleration>
      </actual>
    </state>
  </states>
</trajectory>
